Pico Osu! Code Explanation

This page provides a detailed breakdown of the PICO-8 Lua code for a rhythm game inspired by OSU!.

Introduction

PICO OSU is a rhythm game implemented in PICO-8 Lua, a fantasy console. The game features hit circles appearing on screen that players must click in time with the music. It includes various game states (main menu, song select, playing, paused, results, settings), score tracking, accuracy calculation, and adjustable difficulty settings (OD and AR).

The code is structured into functions that handle initialization, game logic updates, and drawing for each state. It also includes utility functions for input handling, number sanitization, and UI elements.

Utility Functions

These functions provide foundational utilities used throughout the game, from input handling to numerical safety.

Button Release Detection (`btnr`)

function btnr(b)
  return _prev_btn_state[b] and not btn(b)
end

The btnr(b) function detects if a button b was just released. It checks if the button was pressed in the previous frame (_prev_btn_state[b] is true) and is currently not pressed (not btn(b)). This is crucial for single-press actions, preventing multiple triggers from a single long press.

Number Sanitization (`is_nan`, `is_inf`, `sanitize_pico8_int`)

function is_nan(x)
  return x ~= x
end

function is_inf(x)
  return x == 1/0 or x == -1/0
end

function sanitize_pico8_int(val)
    if is_nan(val) or is_inf(val) then
        return 0
    end
    val = flr(val) 
    if val > 32767 then return 32767 end
    if val < -32768 then return -32768 end
    return val
end

PICO-8 has specific integer limits ($-32768$ to $32767$). These functions ensure that numerical values, especially those derived from calculations like scores or accuracy, stay within these bounds and handle potential NaN (Not a Number) or Infinity values, returning 0 in such cases. This prevents unexpected behavior or crashes.

Deep Copy for Tables (`deepcopy`)

function deepcopy(orig)
    local orig_type = type(orig)
    local copy
    if orig_type == 'table' then
        copy = {}
        for k, v in pairs(orig) do
            copy[k] = deepcopy(v)
        end
    else
        copy = orig
    end
    return copy
end

This function creates a complete, independent copy of a Lua table, including nested tables. This is important when you want to modify a table (like the beatmap for a specific song) without affecting the original data structure.

The load_marshmary_data() function directly manipulates PICO-8's memory using poke() calls. In PICO-8, poke(address, value) writes a byte to a specific memory address. These particular poke calls are used to load custom instrument and music pattern data into memory, specifically for the "marshmary" song. This allows the game to play custom sounds and music that are not part of PICO-8's default sound chip capabilities. Address 0x3103, for instance, is related to the music playback state.

PICO-8 games operate on a simple loop: _update() for game logic and _draw() for rendering. These functions act as a dispatcher, calling the appropriate update_X() and draw_X() functions based on the current game_state. This modular approach keeps the code organized. The _draw() function also draws the mouse cursor as a small circle.

Update Play (`update_play`) - Core Logic

function update_play()
  if not song_started then music(current_song.music_pattern,100); song_started=true end;
  if stat(30) and stat(31)=="`" then game_state="paused"; sfx(2); music(-1); return end; -- Pause game

  local ct=time()-start_time; -- Current time in song

  -- Spawn notes
  if beatmap_index<=#beatmap and ct>=beatmap[beatmap_index].sec-preempt_time then
    local n=beatmap[beatmap_index];
    add(active_notes,{x=n.x,y=n.y,hit_time=n.sec,radius=10,judged=false,number=beatmap_index});
    beatmap_index+=1
  end;

  -- Handle player input (click)
  if btnp(4) or btnp(5) then 
    local best_n,best_d,mx,my=nil,1,stat(32),stat(33); -- Mouse x, y
    for n in all(active_notes) do
      if not n.judged and abs(n.hit_time-ct)n.hit_time+(hit_window_ms_50/1000) then
      n.judged=true; combo=0;count_miss+=1;sfx(1);add_feedback("miss",n.x,n.y,8, 30);
      update_accuracy();del(active_notes,n)
    end
  end;

  -- Update feedback popups
  for p in all(feedback_popups) do
    p.y-=0.5; p.life-=1;
    if p.life<=0 then del(feedback_popups,p) end
  end;

  -- Check for map completion
  if(beatmap_index>#beatmap and #active_notes==0) then
    final_grade=calculate_grade()
    final_ur_int=sanitize_pico8_int(calculate_ur())
    game_state="results"; music(-1) 
  end
end

This is the heart of the game. It handles:

  • Starting music.
  • Pausing the game.
  • Spawning new notes based on the current song time and preempt_time.
  • Detecting player clicks (btnp(4) or btnp(5) for Z/X buttons, or mouse clicks).
  • Calculating hit accuracy based on hit_window_ms_300, 100, and 50, and mouse position relative to the note.
  • Updating score, combo, and hit counts (300, 100, 50, miss).
  • Adding visual feedback popups ("300", "miss", etc.).
  • Tracking hit errors for "Unstable Rate" (UR) calculation.
  • Removing notes that have been hit or missed.
  • Transitioning to the results screen when the beatmap is finished.

function draw_play()

Accuracy and Unstable Rate Calculation

function update_accuracy()
  local t_total = count_300 + count_100 + count_50 + count_miss;
  if t_total == 0 then accuracy_int = 100; return end;

  local w_weighted_score = (count_300 * point_300) + (count_100 * point_100) + (count_50 * point_50);
  w_weighted_score = sanitize_pico8_int(w_weighted_score)

  local max_possible_w_score = t_total * point_300;
  max_possible_w_score = sanitize_pico8_int(max_possible_w_score)

  local calculated_acc_int = 0
  if max_possible_w_score > 0 then
      calculated_acc_int = flr((w_weighted_score / max_possible_w_score) * 100);
  end

  accuracy_int = max(0, min(100, sanitize_pico8_int(calculated_acc_int)))
end

function calculate_ur()
  local n = #hit_errors;
  if n == 0 then return 0 end;

  local sum_abs_errors_scaled_ms = 0;
  for e_scaled_ms in all(hit_errors) do
    sum_abs_errors_scaled_ms += abs(e_scaled_ms)
    sum_abs_errors_scaled_ms = sanitize_pico8_int(sum_abs_errors_scaled_ms)
  end;

  local aae_scaled_ms_int = 0;
  if n > 0 then
      aae_scaled_ms_int = flr(sum_abs_errors_scaled_ms / n);
  end
  return aae_scaled_ms_int 
end

update_accuracy() calculates the player's current accuracy as a percentage. It uses a weighted sum of 300s, 100s, and 50s against the maximum possible score for the notes hit so far. calculate_ur() computes the "Unstable Rate" (UR), which is a common metric in rhythm games indicating the consistency of a player's timing. It's the average absolute error of all hits, scaled to milliseconds. A lower UR indicates more consistent timing.
